# ICS3U – Culminating Performance Task (CPT)
### Interactive Application using Java & Processing


## Overview

This Culminating Performance Task (CPT) is your opportunity to demonstrate your understanding of the programming concepts covered in ICS3U by designing and building an **interactive application** using Java with the Processing library.

Your program does not have to be a game, although games are allowed. Any interactive program that allows a user to meaningfully interact with logic, visuals, and state is acceptable.

This is a major assessment and represents a significant portion of your final course grade.



## Learning Goals

Through this project, you will demonstrate your ability to:

- design and implement an interactive program
- apply programming concepts in meaningful ways
- structure and organize a non-trivial codebase
- explain and defend your design and implementation decisions


## Project Requirements

Your project must be an **interactive Processing sketch** written in Java and must clearly demonstrate effective use of the programming concepts learned in this course.

### Programming Concepts Expected

Your project should demonstrate **effective use** of the following concepts **somewhere in the codebase**:

- variables and data types  
- expressions and calculations  
- user input and interactivity (keyboard and/or mouse)  
- conditional logic  
- repetition (loops)  
- methods and decomposition  
- arrays and/or ArrayLists  

These concepts should be used **purposefully** and **appropriately** as part of your program’s logic. Adding features only to “check a box” is discouraged.



## Project Scope & Constraints

To keep projects comparable and manageable:

- You must use **Java with the Processing library**
- Your program must run from `src/Sketch.java`
- Do **not** use external libraries beyond Processing
- Your project should be understandable by someone reading your code for the first time
- Your application must include **meaningful user interaction**



## Group Work

You may complete this project:

- **individually**, or  
- **in a pair (maximum two students)**

If working in a pair:
- You are expected to work together using [pair programming](https://youtu.be/nWm79e9uH5k)

## Use of Graphic & Media Assets

Any graphic, audio, or other media assets used in your project must be **one of the following**:

- Original work created by you, or  
- Public domain, or  
- Creative Commons licensed (e.g. CC BY, CC BY-SA), or
- AI-generated (with disclosure — see below)

If you use assets that are **not your own**, you must:

- ensure the license allows reuse in a school project
- provide **proper attribution**

### Image Guidelines

Keep image files small. Large images cause problems with GitHub and slow down your repository.

- Individual images: aim for **under 100 KB**
- Total image assets: keep well **under 1 MB** for the whole project
- Sprites: around **128 × 128 px**
- Sprite sheets: around **256 × 256 px**
- Backgrounds: no larger than your sketch window size

Use **PNG** only when transparency is needed. Use **JPG** for backgrounds and photos.

Avoid using raw screenshots, phone photos, or unresized design exports. If you need to resize an image, use a free browser-based editor such as [Photopea](https://www.photopea.com/).

**Important:** If you add or change image files, commit and push immediately. Do not stack multiple commits involving images before pushing — catch problems early, when they are easy to fix.

### Attribution Requirements

Attribution must appear in an appropriate section of your `README.md` file. Use the formats below depending on the asset type.

**Public Domain**
```
Asset name or description
Source: [link]
License: Public Domain
```

**Creative Commons (e.g. CC BY)**
```
Asset name or description
Author: [name]
Source: [link]
License: CC BY 4.0
```

**AI-Generated**
```
Asset name or description
Generated by: [tool name, e.g. ChatGPT, Midjourney]
Prompted and generated by: [your name]
License: AI-generated content (no external copyright claimed)
```

If you edited or modified an AI-generated asset, note that as well (e.g. "Cropped and colour-adjusted by Student Name").

Do not claim AI-generated assets as hand-drawn or original artwork.

**Avoid these common mistakes:**
- Linking to Google Images instead of the original source
- Writing "Internet" or "ChatGPT" without context
- Missing license information
- Failing to disclose AI use

Failure to properly credit external assets may result in a deduction under professionalism or academic integrity.
